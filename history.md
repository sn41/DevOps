## Курс: Управление жизненным циклом приложения (ALM) в стиле DevOps

Этот цикл лекций проведет слушателей от основ планирования до эксплуатации и мониторинга высоконагруженных систем.



### Лекция 1. Философия ALM и культура DevOps
* **Теория:** Что такое ALM (Application Lifecycle Management) и чем он отличается от SDLC. 
* **DevOps Mindset:** Три пути (The Three Ways) — Поток, Обратная связь, Непрерывное обучение.
* **Процессы:** Обзор этапов: Plan -> Code -> Build -> Test -> Release -> Deploy -> Operate -> Monitor.
* **Инструментарий:** Обзор современного ландшафта (CNCF Landscape).

### Лекция 2. Управление исходным кодом и стратегии ветвления
* **Основы:** Git как фундамент ALM. 
* **Workflow:** Сравнение GitFlow, GitHub Flow и Trunk-Based Development. Почему DevOps любит «короткие» ветки.
* **Code Review:** Как процесс проверки кода влияет на жизненный цикл и качество продукта.
* **Conventional Commits:** Автоматизация чейнджлогов на основе истории коммитов.



### Лекция 3. Continuous Integration (CI): Фундамент качества
* **Автоматизация сборки:** От локального `make` до облачных Runner-ов.
* **Quality Gates:** Интеграция статического анализа кода (SAST), линтеров и проверки зависимостей.
* **Пирамида тестирования:** Unit, Integration, и почему нельзя полагаться только на UI-тесты в CI.
* **Артефакты:** Понятие неизменяемости (Immutability) сборки.

### Лекция 4. Контейнеризация и управление артефактами
* **Docker:** Создание эффективных образов (multi-stage builds).
* **Registry:** Жизненный цикл образов: хранение, тегирование (SemVer) и очистка старых артефактов.
* **Безопасность:** Сканирование образов на уязвимости (Trivy, Grype) перед публикацией.

### Лекция 5. Continuous Delivery (CD): Стратегии развертывания
* **Разница между Delivery и Deployment:** Когда нажимать кнопку «Deploy», а когда автоматизировать всё.
* **Zero Downtime:** Механизмы обновления без простоя.
* **Продвинутые техники:**
    * **Blue-Green Deployment** (переключение трафика).
    * **Canary Releasing** (постепенная раскатка).
    * **Feature Toggles** (управление функционалом в рантайме).



### Лекция 6. Infrastructure as Code (IaC) и GitOps
* **Декларативный подход:** Почему мы описываем инфраструктуру кодом (Terraform, Ansible, CloudFormation).
* **Концепция GitOps:** Git как «единственный источник истины» для состояния кластера (на примере ArgoCD или Flux).
* **Дрифт:** Как бороться с расхождением реальности и конфигурации.

### Лекция 7. Наблюдаемость (Observability) и обратная связь
* **Три столпа:** Метрики, Логи, Трассировка (Metrics, Logs, Traces).
* **SLI/SLO/SLA:** Как понять, что приложение «здорово», и когда нужно прекратить деплоить новые фичи.
* **Feedback Loop:** Автоматические откаты (Rollback) при ухудшении метрик после деплоя.

### Лекция 8. DevSecOps: Безопасность на каждом этапе
* **Shift Left:** Почему безопасность начинается с IDE, а не с фаервола.
* **Secrets Management:** Как не «закоммитить» пароли (Vault, SOPS, Secrets Store CSI).
* **Compliance as Code:** Автоматическая проверка соответствия стандартам безопасности в пайплайне.
* **Итоги курса:** Построение финальной карты идеального ALM-процесса.
